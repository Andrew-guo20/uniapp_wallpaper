### 1.1 配置 uni-id（0.5 天）

**任务清单：**
- [ ] 在 `uniCloudWallPaper/uni_modules/uni-config-center/` 下创建 `uni-id/config.json`
- [ ] 配置微信小程序登录（`wxedf7c9be266ef39c`）
- [ ] 配置 token 有效期、密码强度规则
- [ ] 上传部署 uni-id-common 云函数公共模块
- [ ] 验证 uni-id 服务可用

**配置文件要点：**
```json
{
  "passwordSecret": "auto-generated",
  "tokenSecret": "auto-generated",
  "tokenExpiresIn": 7200,
  "mp-weixin": {
    "oauth": {
      "weixin": {
        "appid": "wxedf7c9be266ef39c",
        "appsecret": "<微信小程序AppSecret>"
      }
    }
  }
}
```

**⚠️ 注意：** 需在微信公众平台获取 AppSecret，配置到 uni Cloud 控制台的环境变量而非硬编码。

### 1.2 前端登录流程（1 天）

**新增文件：** `wallpaper/pages/login/login.vue`

**改动文件：**
- `wallpaper/App.vue` — 启动时检查登录状态
- `wallpaper/API/apis.js` — 新增 `apiLogin()`、`apiGetUserProfile()`
- `wallpaper/utils/` — 新增 `auth.js`（token 管理工具）

**实现要点：**
- 微信一键登录：`uni.login()` → 获取 code → 调用 uni-id 登录接口 → 返回 token
- Token 持久化：`uni.setStorageSync('uni_id_token', token)`
- 全局状态管理：使用 `reactive` 管理用户登录态
- 游客模式：不登录可浏览壁纸，收藏/评论/投稿需登录

**验收标准：**
- 点击微信登录 → < 5 秒进入首页
- Token 过期自动刷新
- 退出登录后恢复游客模式

### 1.3 deviceId → uid 数据迁移（0.5 天）

**改动文件：** `index.obj.js`（云对象）

**迁移策略：**
```
1. 用户首次微信登录 → 获取 uid
2. 后台异步任务：查找该 deviceId 的历史评分/下载记录
3. 将记录的 deviceId 字段更新为 uid
4. 新产生数据直接使用 uid
```

**向后兼容：**
```javascript
// _before 钩子改造
_before: function () {
  const clientInfo = this.getClientInfo()
  this.uid = clientInfo.uid || clientInfo.deviceId || 'anonymous'
  this.deviceId = clientInfo.deviceId || 'anonymous' // 保留用于迁移
}
```

**验收标准：**
- 老用户数据 100% 迁移，无丢失
- 新用户评分/下载直接关联 uid
- 迁移脚本可重复执行，幂等

### 3.3 个性化推荐（1 天）

**推荐算法（协同过滤 + 标签权重）：**

```
1. 收集用户行为：浏览（+1分）、下载（+3分）、评分高分（+2分）、收藏（+3分）
2. 提取行为壁纸的标签，累加权重
3. 按权重排序取 TOP 10 标签
4. 查询这些标签下已发布壁纸，按权重 + 评分降序排列
5. 排除用户已浏览/下载过的壁纸
6. 不足 9 张则用高评分壁纸补齐
```

**新增方法：**
```javascript
getRecommendWalls({ uid, pageSize = 9 })
// 如果用户无历史数据，返回高评分随机壁纸（冷启动）
// 返回格式与 getRandomWall 一致，复用现有 UI
```

**前端接入：**
- 首页"每日推荐"区域改为"为你推荐"
- 混合策略：有推荐数据 → 展示推荐；无数据 → 展示随机热门

### 4.1 uni-push 消息推送（0.5 天）

**推送场景：**
| 场景 | 触发时机 | 推送内容 |
|------|---------|---------|
| 投稿通过 | 管理员审核通过 | "你的壁纸《xxx》已通过审核" |
| 投稿拒绝 | 管理员审核拒绝 | "你的壁纸《xxx》未通过审核，原因：xxx" |
| 新壁纸上架 | 定时任务完成同步 | "本周新增 20 张精美壁纸，快来看看" |
| 系统通知 | 管理员发布公告 | 公告标题 + 摘要 |

**技术实现：**
- 使用 uni-push 2.0（基于 uniapp 统一推送）
- 云对象封装推送方法：`sendPush({ uid, title, content, payload })`
- 前端客户端监听 `onPushMessage`，点击通知跳转对应页面

### 4.2 云函数定时触发器（0.5 天）

**定时任务：**
| 任务 | 频率 | 说明 |

```json
  }
}
```

## 云对象模块化方案（贯穿各阶段）

当前 `index.obj.js`（868 行单文件）拆分为：

```
uniCloud-alipay/cloudfunctions/
├── wallpaper/                    # 用户端云对象
│   ├── index.obj.js              #   主入口 + _before + mergeData
│   ├── wallpaper.js              #   getBanner/getRandomWall/getClassify/...
│   ├── user.js                   #   getUserInfo/getUserWallList
│   ├── favorite.js               #   addFavorite/removeFavorite/...     [NEW]
│   ├── comment.js                #   addComment/getComments/...         [NEW]
│   ├── upload.js                 #   submitUpload/getMyUploads          [NEW]
│   ├── recommend.js              #   getRecommendWalls                  [NEW]
│   └── package.json
│
├── wallpaper-admin/              # 管理端云对象 [SPLIT FROM wallpaper]
│   ├── index.obj.js              #   主入口 + 管理员权限校验
│   ├── banner-admin.js
│   ├── classify-admin.js
│   ├── wallpaper-admin.js
│   ├── news-admin.js
│   ├── comment-admin.js          #   [NEW]
│   ├── user-admin.js             #   [NEW]
│   ├── upload-review.js          #   [NEW]
│   ├── stats.js                  #   [NEW]
│   └── package.json
│
└── common/                       # 公共模块
    ├── utils.js                  #   mergeData、权限校验、敏感词过滤
    ├── db.js                     #   数据库连接单例
    └── package.json
```

**模块化原则：**
- 每个模块文件导出一个对象，包含相关方法
- `index.obj.js` 中 `Object.assign(this, module1, module2, ...)` 合并
- 公共逻辑抽取到 `common/utils.js`
