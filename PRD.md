# 一壁之力 — 产品需求文档 (PRD)

> **版本**: 2.0.0 | **日期**: 2025-06-25 | **状态**: 已评审
> **作者**: 产品团队 | **技术栈**: uni-app Vue3 + uniCloud 支付宝云

## 1. 执行摘要

### 问题陈述
一壁之力小程序 MVP 已完成基础浏览、评分和下载功能（203 张壁纸、10 个分类）。但缺乏用户体系、可视化后台管理、以及提升用户粘性的互动功能。当前用设备 ID（deviceId）做临时用户标识，无法追踪用户行为、沉淀用户关系。

### 解决方案
在现有 MVP 基础上，接入 uni-id 微信登录体系，搭建 uni-admin 可视化管理后台，新增收藏、评论、个性化推荐、用户投稿等增强功能。从"能用的工具"进化为"可运营的产品"。

### 业务影响
- 用户留存率提升：从当前匿名模式到有账户体系，预计留存提升 40%+
- 运营效率提升：可视化管理后台替代手动 API 调用，日常运营效率提升 80%+
- 内容生态扩展：用户投稿机制让内容从单向供给变为双向生态

### 成功指标
| 指标 | 当前值 | 目标值（6个月） |
|------|--------|-----------------|
| 微信登录率 | 0% | > 60% |
| 用户 7 日留存 | 无数据 | > 25% |
| 收藏功能使用率 | 无 | > 30% |
| 评论参与率 | 无 | > 10% |
| 管理后台操作覆盖率 | 0%（纯 API） | 100% 日常操作 |

## 2. 问题定义

### 2.1 用户问题

**目标用户画像：**

| 画像 | 描述 | 核心需求 |
|------|------|---------|
| **手机美化爱好者**（18-30岁） | 追求个性化手机桌面，经常换壁纸 | 海量高清壁纸、便捷下载、分类浏览 |
| **设计师/创作者**（20-35岁） | 对视觉质量要求高，偶尔也愿分享作品 | 高清原图、评分参考、投稿渠道 |
| **小程序运营者**（管理员） | 负责内容更新、审核、数据分析 | 可视化管理、批量操作、数据统计 |

**核心痛点：**
- 用户看了壁纸无法收藏，下次就找不到
- 不知道壁纸质量如何，只能一张张下载试
- 内容更新依赖技术爬虫，缺少用户共创
- 管理员维护数据需写代码/调 API，效率极低

### 2.2 市场机会

微信小程序生态中壁纸类应用竞争激烈，但高质量、有用户体系的精品壁纸小程序仍有空间。核心差异化在于：
- **uniCloud 自建后端**：数据自主可控，不依赖第三方 API
- **用户共创生态**：评分 + 评论 + 投稿形成内容飞轮
- **个性化推荐**：基于用户行为的智能推荐，提升发现效率

### 2.3 不做（Out of Scope）

- 视频壁纸 / Live Photo（本期不做）
- 付费会员体系（P3 远期）
- 社交功能（关注/私信）
- 多语言国际化
- 桌面端 Web 应用

### 7.2 uni-id 集成要点

- **配置文件**: `uniCloud-alipay/uni_modules/uni-config-center/uni-id/config.json`
- **登录方式**: 微信小程序一键登录 (`uni.login` + `uni-id-pages`)
- **Token 管理**: uni-id-common 自动处理，前端无需手动管理
- **迁移策略**: `_before` 钩子中 `this.uid = clientInfo.uid || clientInfo.deviceId` 向后兼容
- **管理权限**: uni-admin 自动关联管理员角色，云对象方法通过 `uni-id` 校验权限

### 7.4 安全设计

- **云对象权限**：`_before` 中校验 `uid`，未登录用户只读
- **管理操作**：管理员身份通过 uni-id 角色校验，敏感操作二次确认
- **内容安全**：评论/投稿内容接入微信内容安全审核（`security.msgSecCheck`）
- **频率限制**：评论/投稿接口加入频率限制（每用户每分钟最多 5 条）
- **云存储**：用户投稿图片设置私有权限，审核通过后改为公共读

## 8. 非功能性需求

| 维度 | 要求 |
|------|------|
| **性能** | 首页加载 < 2s，列表分页 < 1s，图片懒加载 |

## 9. 风险与缓解

| 风险 | 概率 | 影响 | 缓解策略 |

| deviceId → uid 数据迁移失败 | 低 | 高 | 分批迁移 + 回滚方案 + 数据备份 |
| 支付宝云服务不稳定 | 低 | 中 | 关键数据本地备份，多供应商评估 |
| uni-admin 功能不够用 | 中 | 中 | 保留云对象 API，可自建定制后台 |
| 用户生成内容违规风险 | 中 | 高 | 接入微信内容安全 + 先审后发 + 人工复核 |
| 云对象体积过大性能差 | 中 | 低 | 模块化拆分，按功能独立部署 |

### B. 微信小程序审核注意事项

- 服务类目：工具 → 图片/壁纸
- 需要用户隐私保护指引
- 用户生成内容（评论/投稿）需有举报和审核机制
- 登录功能需提供游客模式（不登录也能浏览）
