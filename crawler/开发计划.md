## 当前能力

## 待开发任务

### 任务 1：配置化 API Key 管理（阶段四，0.25 天）

**现状**：Unsplash/Pexels API Key 硬编码在 `config.py`
**目标**：支持环境变量或 `.env` 文件

- [ ] 读取 `.env` 文件中的 API Key
- [ ] 未配置 Key 时自动跳过对应源（不报错）
- [ ] 添加 `.env.example` 模板文件
- [ ] `.env` 加入 `.gitignore`

### 任务 2：定时同步集成（阶段四，0.5 天）

**现状**：手动 `python main.py` 触发
**目标**：支持被云函数定时触发器调用

- [ ] 新增 `--headless` 模式（无交互、静默运行）
- [ ] 同步结果输出 JSON（成功数/失败数/新增数）
- [ ] 失败重试 + 错误日志文件
- [ ] 云对象 URL 化地址改为可配置环境变量
- [ ] 接入云函数 `_timing` 定时触发器（每周一 02:00）

### 任务 3：数据源健康检查（阶段四，0.25 天）

- [ ] `python main.py --health-check` 检查所有源是否可用
- [ ] 输出各源：可用状态 / 今日还可调用次数 / 响应延迟
- [ ] 告警：源不可用时输出错误详情

### 任务 4：壁纸去重增强（阶段三，0.25 天）

**现状**：按文件名本地去重
**目标**：云端去重（查询数据库是否已存在相同 URL）

- [ ] 导入前调用云对象检查 `picurl` 是否重复
- [ ] 跳过已存在的壁纸，不重复上传 + 导入
- [ ] 日志输出去重跳过的数量

### 任务 5：搜索引擎优化（阶段三，0.25 天）

**现状**：固定搜索词列表
**目标**：动态调整搜索策略

- [ ] 支持从数据库读取各分类已有壁纸数
- [ ] 壁纸数少的分类自动增加采集量
- [ ] 支持配置目标每个分类壁纸数（默认 200）
