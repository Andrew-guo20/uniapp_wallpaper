## 阶段一：健壮性增强（0.5 天）

> **目标**：配置安全 + 云端去重，提升采集可靠性

### 1.1 API Key 配置化（0.25 天）

**现状**：Unsplash / Pexels API Key 硬编码在 `config.py`
**目标**：环境变量管理，未配置自动跳过

- [x] 读取 `.env` 文件（`python-dotenv` 或手动解析）
- [x] `UNSPLASH_ACCESS_KEY` / `PEXELS_API_KEY` 环境变量
- [x] 未配置 Key 时输出警告 + 自动跳过该源
- [x] 创建 `.env.example` 模板文件
- [x] `.env` 加入 `.gitignore`

### 1.2 云端去重（0.25 天）

**现状**：仅本地文件名去重
**目标**：导入前查数据库是否已有相同 `picurl`

- [x] 导入前调云对象方法检查 `picurl` 是否存在
- [x] 已存在的壁纸跳过上传 + 导入
- [x] 日志输出去重跳过的数量（`Skipped: 5 already exist`）

---

## 阶段二：自动化集成（0.5 天）

> **目标**：支持被定时触发器自动调用

### 2.1 无交互模式（0.25 天）

- [x] 新增 `--headless` 参数：静默运行、无交互确认
- [x] 同步结果输出结构化 JSON（成功数/失败数/新增数/去重跳过数）
- [x] 失败重试（单张壁纸 3 次，指数退避）
- [x] 错误日志写入文件 `logs/crawl_YYYYMMDD.log`

### 2.2 云对象 URL 配置化（0.25 天）

- [x] `WALLPAPER_CLOUD_URL` 环境变量配置云对象地址
- [x] 连接测试：启动时检查云对象连通性
- [x] 接入云函数 `_timing` 定时触发器（每周一 02:00 UTC+8）
- [x] 编写触发器调用的云对象方法 `triggerCrawl()`

---

## 阶段三：可观测性（0.5 天）

> **目标**：知道采集器是否正常、数据是否健康

### 3.1 数据源健康检查（0.25 天）

- [x] `python main.py --health-check`
- [x] 逐源检测：可用状态 / 今日剩余调用次数 / 响应延迟
- [x] 不可用时输出详细错误信息

### 3.2 搜索策略优化（0.25 天）

- [x] 从数据库读取各分类已有壁纸数
- [x] 壁纸少的分类自动增加 `-n` 采集量
- [x] 支持 `--target-per-category 200` 参数
- [x] 输出各分类壁纸数统计
