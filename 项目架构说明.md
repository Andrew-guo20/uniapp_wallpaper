# uniapp_wallpaper 项目架构说明

## 项目总览

```
uniapp_wallpaper/
├── wallpaper/          # 前端：uni-app Vue3 壁纸小程序
├── uniCloudWallPaper/  # 后端：uniCloud 云服务项目
├── crawler/            # 工具：Python 壁纸数据采集器
├── uniCloud-开发计划.md  # 开发计划文档
├── uniapp开发.txt       # uni-app 学习笔记
└── 开发uniCloud项目.txt  # uniCloud 入门笔记
```

三个子项目各司其职，共同构成完整的壁纸小程序系统。

## 一、wallpaper — 前端小程序

### 定位
面向用户的 uni-app Vue3 壁纸小程序，支持 H5、微信小程序、支付宝小程序等多端运行。

### 目录结构

```
wallpaper/
├── API/apis.js                   # 统一 API 层（12 个接口）

│   ├── classlist/classlist.vue   #   列表页（分类下壁纸+用户下载/评分历史）

### 核心机制

#### 数据流
```
页面 → API/apis.js → uniCloud.importObject('wallpaper') → 云对象 → 数据库
```

所有页面通过 `API/apis.js` 统一调用后端，不直接操作数据库。页面代码无需关心数据来源（此前是第三方 HTTP API，现已切换为 uniCloud 云对象）。

#### 12 个 API 函数

| 函数 | 对应云对象方法 | 用途 | 调用页面 |
|------|--------------|------|---------|
| `apiGetBanner()` | `getBanner()` | 获取轮播图 | 首页 |
| `apiGetDayRandom()` | `getRandomWall()` | 每日推荐 9 张 | 首页 |
| `apiGetClassify()` | `getClassify()` | 分类列表 | 首页、分类页 |
| `apiGetClassifyDetail()` | `getWallList()` | 分类下壁纸分页 | 列表页 |
| `apiDetailWall()` | `getDetailWall()` | 单个壁纸详情 | 预览页(分享) |
| `apiGetsetupScore()` | `setupScore()` | 用户评分 | 预览页 |
| `apiWriteDownload()` | `downloadWall()` | 记录下载 | 预览页 |
| `apiUserInfo()` | `getUserInfo()` | 用户统计信息 | 用户中心 |
| `apiGetHistoryList()` | `getUserWallList()` | 下载/评分历史 | 列表页 |
| `apiGetNotice()` | `getNewsList()` | 公告列表 | 首页 |
| `apiNoticeDetail()` | `getNewsDetail()` | 公告详情 | 公告详情页 |
| `apiSearchData()` | `searchWall()` | 搜索壁纸 | 搜索页 |

#### 页面间数据传递
- **首页 → 预览**：通过 `uni.setStorageSync('storageClassList', list)` 将当前壁纸列表存入本地缓存，预览页读取后展示轮播
- **列表页 → 预览**：同上机制
- **分享链接**：URL 带 `id` 参数，预览页通过 `apiDetailWall({id})` 单独查询

## 二、uniCloudWallPaper — 云服务后端

### 定位
uniCloud 服务端项目，包含云对象和数据库 Schema。负责所有业务逻辑和数据存取。

### 目录结构

```
uniCloudWallPaper/
├── uniCloud-alipay/
│   ├── cloudfunctions/wallpaper/
│   │   ├── index.obj.js          # 云对象（32 个方法，约 800 行）
│   │   └── package.json          # 云函数配置（超时 30s）
│   └── database/
│       ├── wallpaper-banner.schema.json       # 轮播图表
│       ├── wallpaper-classify.schema.json     # 分类表
│       ├── wallpaper-list.schema.json         # 壁纸详情表（核心）
│       ├── wallpaper-news.schema.json         # 公告表
│       ├── wallpaper-user-score.schema.json   # 用户评分表
│       ├── wallpaper-download-record.schema.json  # 下载记录表
│       └── JQL查询.jql                        # JQL 调试脚本
├── uni_modules/
│   ├── uni-config-center/        # uni-id 配置中心（预留）
│   └── uni-id-common/            # uni-id 公共库（预留）
├── pages/index/index.vue         # 默认首页（占位）
└── manifest.json                 # 项目配置
```

### 数据库设计（6 张表）

```
wallpaper-classify (分类)
    │ 1:N (classid)
    ▼
wallpaper-list (壁纸详情，核心表)
    │ 1:N (wallId)
    ├── wallpaper-user-score (评分记录)
    └── wallpaper-download-record (下载记录)

wallpaper-banner (轮播图) —— 独立
wallpaper-news (公告) —— 独立
```

#### wallpaper-list（壁纸详情表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | string | 系统自动生成 |
| `classid` | string | 关联分类 ID |
| `smallPicurl` | string | 缩略图云存储 URL |
| `picurl` | string | 原图云存储 URL |
| `nickname` | string | 发布者昵称 |
| `score` | double | 综合评分（0-5，支持半分） |
| `description` | string | 壁纸简介 |
| `tabs` | array | 标签数组 |
| `downloadCount` | int | 累计下载次数 |
| `scoreCount` | int | 评分人数 |
| `create_time` | timestamp | 上传时间 |
| `status` | int | 0=待审核 1=已发布 2=已下架 |

索引：`idx_classid_status`（classid+status）、`idx_status_create`（status+create_time）

#### 权限策略

| 表 | read | create | update | delete | 说明 |
|----|------|--------|--------|--------|------|
| banner/classify/list/news | ✅ | ❌ | ❌ | ❌ | 内容表：用户只读 |
| user-score | ✅ | ✅ | ❌ | ❌ | 评分：可读可写 |
| download-record | ✅ | ✅ | ❌ | ❌ | 下载记录：可读可写 |

- 前端仅能读取内容表、写入评分和下载记录
- 云对象使用传统数据库模式（`db`），不受 Schema 权限限制，可执行任意 CRUD

### 云对象方法（32 个）

#### 前端调用方法（12 个）

| 方法 | 功能 | 关键技术点 |
|------|------|-----------|
| `getBanner()` | 轮播图列表 | `orderBy('sort', 'asc')` |
| `getRandomWall()` | 每日推荐 9 张 | 随机 skip + `limit(9)` |
| `getClassify(data)` | 分类列表 | 支持 `pageSize` 限制 |
| `getWallList(data)` | 分类下壁纸分页 | `where({classid, status:1})` + skip/limit |
| `getDetailWall(data)` | 壁纸详情 | `where({_id})` 返回数组 |
| `setupScore(data)` | 用户评分 | 唯一索引防重复 + 原子更新平均分 |
| `downloadWall(data)` | 记录下载 | `add` + `dbCmd.inc(1)` 原子计数 |
| `getUserInfo(data)` | 用户统计 | Promise.all 并行查询 |
| `getUserWallList(data)` | 用户历史 | `aggregate().lookup()` 联表 |
| `getNewsList(data)` | 公告列表 | 支持 `select:true` 仅置顶 |
| `getNewsDetail(data)` | 公告详情 | `get` + `dbCmd.inc(1)` 浏览计数 |
| `searchWall(data)` | 搜索壁纸 | JS 过滤（支付宝云不支持 dbCmd.regex） |

#### 后台管理方法（20 个）

- **轮播图**：adminGetBanners / adminCreateBanner / adminUpdateBanner / adminDeleteBanner
- **分类**：adminGetClassifies / adminCreateClassify / adminUpdateClassify / adminDeleteClassify（含关联壁纸检查）
- **壁纸**：adminGetWalls / adminCreateWall / adminUpdateWall / adminDeleteWall（级联删除评分+下载记录）/ adminReviewWall（批量审核）/ adminGetWallStats（统计）/ adminBatchImport（批量导入，供爬虫调用）/ adminUpload（上传到云存储）
- **公告**：adminGetNews / adminCreateNews / adminUpdateNews / adminDeleteNews

### 关键架构决策

1. **全部使用传统数据库模式（db + dbCmd）**，不使用 JQL
   - JQL 受 Schema 权限限制（create:false 会阻止云对象写入）
   - JQL 不支持 `dbCmd.inc()` / `aggregate()` 等高级操作

2. **暂不接入 uni-id 用户体系**，使用 `deviceId` 作为用户标识
   - 后续迁移只需将 `deviceId` 替换为 `openid`

3. **HTTP URL 化兼容处理**
   - `mergeData()` 函数合并 query string 和 POST JSON body 参数
   - 支付宝云不支持 `dbCmd.regex`，搜索改用 JS 过滤

## 三、crawler — 壁纸数据采集器

### 定位
Python 命令行工具，从多个壁纸源自动采集、处理并导入壁纸数据。

### 目录结构

```
crawler/
├── main.py                 # 主流程编排（命令行入口）
├── config.py               # 全局配置（路径、API Key、URL）
├── classify_map.py         # 分类映射表（10 个分类 + 搜索词）
├── downloader.py           # 图片下载模块（支持重试）
├── thumbnail.py            # 缩略图生成模块（Pillow, 300px webp）
├── uploader.py             # 云存储上传模块（base64 → adminUpload）
├── importer.py             # 批量导入模块（adminBatchImport / JSON 导出）
├── requirements.txt        # Python 依赖
├── sources/                # 壁纸数据源
│   ├── __init__.py         #   BaseSource 抽象基类
│   ├── bing.py             #   Bing 每日壁纸（免费，无需 Key）
│   ├── wallhaven.py        #   Wallhaven API（免费，无需 Key）
│   ├── unsplash.py         #   Unsplash API（需 Access Key）
│   └── pexels.py           #   Pexels API（需 API Key）
├── downloads/              # 本地图片存储（按分类存放）
│   ├── 风景/originals/     #   原图
│   ├── 风景/thumbnails/    #   缩略图
│   └── ...
└── exports/                # JSON 导出目录（离线模式）
```

### 五阶段流水线

```
main.py → sources → downloader → thumbnail → uploader → importer
 (编排)    (采集)     (下载)       (缩略图)     (上传)      (导入)
```

#### 阶段 1：采集（sources/）

4 个数据源模块，每个实现 `search()` 方法和 `search_all_pages()` 自动翻页：

| 数据源 | API | 需要 Key | 每次返回 |
|--------|-----|---------|---------|
| **Bing** | HPImageArchive | ❌ 免费 | 8 张/天 |
| **Wallhaven** | api/v1/search | ❌ 免费 | 24 张/页 |
| **Unsplash** | api.unsplash.com | ✅ Access Key | 30 张/页 |
| **Pexels** | api.pexels.com | ✅ API Key | 80 张/页 |

中文分类名通过 `KEYWORD_MAP` 映射到对应的英文搜索词。

#### 阶段 2：下载（downloader.py）

- 流式下载，支持断点续传检测
- 3 次重试 + 指数退避
- 自动去重（已存在文件跳过）
- 按分类存入 `downloads/{分类}/originals/`

#### 阶段 3：缩略图（thumbnail.py）

- Pillow 缩放至 300px 宽，等比高度
- WebP 格式，80% 质量
- 自动处理 RGBA/CMYK 等特殊模式
- 存入 `downloads/{分类}/thumbnails/xxx_small.webp`

#### 阶段 4：上传（uploader.py）

- 读取本地文件 → base64 编码
- 调用 `adminUpload(fileName, base64)` 上传到 uniCloud 云存储
- 返回云端 URL

#### 阶段 5：导入（importer.py）

- 每批 50 条调用 `adminBatchImport({walls: [...]})`
- 支持离线模式：导出 JSON 文件供手动导入

### 命令行用法

```bash
# 查看帮助
python main.py --help

# 列出所有分类/数据源

# 本地测试（只下载+缩略图，不上传）

## 五、uni-app 核心技术笔记

> 以下内容提取自 `uniapp开发.txt` 和 `开发uniCloud项目.txt`，记录开发过程中的关键知识点。

### 5.1 项目创建与配置

**创建 uniCloud 项目：** HBuilderX 新建 uni-app 项目时勾选"启用 uniCloud"，选择服务供应商（支付宝云 / 阿里云 / 腾讯云）。

**关联服务空间：** 项目右键 → 关联 uniCloud 服务空间 → 选择已创建的空间。

**创建云对象：** `uniCloud/cloudfunctions` 目录右键 → 新建云函数/云对象（勾选云对象）→ 生成 `index.obj.js` 和 `package.json`。右键运行可自动生成 `同名.param.js` 调试文件。

### 5.2 前后端分离模式

```
前端（客户端）         后端（云对象）
─────────────        ─────────────
只做展示和交互    →    操作数据库
不直接操作数据库   ←    提供接口返回数据
```

- 数据操作全部在服务端（云对象）完成，客户端不直接操作数据库
- 前端通过 `const obj = uniCloud.importObject("name")` 导入云对象，调用其方法
- 云对象方法返回 `{errCode: 0, data: ...}` 统一格式

### 5.3 数据库查询两种模式对比

| 特性 | JQL 模式 | 传统模式（db） |
|------|---------|-------------|
| 受 Schema 权限限制 | ✅ 是 | ❌ 否 |
| `dbCmd.inc()` 原子操作 | ❌ 不支持 | ✅ 支持 |
| `aggregate().lookup()` 联表 | ❌ 部分支持 | ✅ 支持 |
| `dbCmd.regex()` 正则搜索 | 视平台而定 | ✅ 视平台而定 |
| 更新数组指定元素 | ❌ 不支持 | ✅ 支持 |
| 适用场景 | 客户端简单查询 | 服务端复杂业务 |

**本项目选择传统模式的原因：** 云对象是服务端代码，应拥有完整数据库权限；需要用到 `inc()` / `aggregate()` / `lookup()` 等高级操作。

### 5.4 uniCloud-db 组件（客户端查询）

```vue
<unicloud-db collection="demo-wallpaper,demo-classify" v-slot:default="{data, error, loading, hasMore}">
  <!-- data: 查询结果, error: 错误, loading: 加载中, hasMore: 是否有更多 -->
</unicloud-db>
```

也可用 `colList` 数组替代逗号分隔的 collection 字符串。支持 `field` 指定返回字段。受 Schema 权限约束。

### 5.5 云函数生命周期

| 处理器 | 说明 |
|--------|------|
| `_before` | 前置拦截器，用于 token 校验、提取客户端信息 |
| `_after(error, result)` | 后置处理器，有 after 时必须 return 结果 |
| `_timing` | 定时触发器，需在 `package.json` 的 `cloudfunction-config` 中配置 |

**云对象特有 API：**
- `this.getClientInfo()` — 获取客户端信息（deviceId、clientPlatform 等）
- `this.getHttpInfo()` — 获取 HTTP 请求信息（URL 化时使用）

### 5.6 云对象 URL 化

**开启方式：** uniCloud 控制台 → 服务空间 → 云函数/云对象 → 函数详情 → 开启 URL 化 → 获得默认长域名。

**调用格式：** `POST https://域名/云对象名/方法名`，参数通过 query string 或 POST JSON body 传递。

**注意事项：**
- body 参数需要手动从 `this.getHttpInfo().body` 解析
- 可通过 apipost 等工具记录和调试接口
- URL 有长度限制，大量数据必须用 POST body

### 5.7 云存储

**上传方式：**
- **客户端直传**（推荐）：`uniCloud.uploadFile()` 直接从客户端上传，不经云函数中转，效率高
- **云函数上传**：在云函数中调用，多一层中转
- 也可用 `uni-file-picker` 组件，但无法实现点击删除同步删除

**关键 API：**

| API | 说明 |
|-----|------|
| `uni.chooseImage()` | 本地选择图片 |
| `uniCloud.uploadFile({cloudPath, fileContent})` | 上传文件，支持单张/批量 |
| `uniCloud.getTempFileURL({fileList})` | fileID 转临时 HTTPS URL |
| `uniCloud.deleteFile({fileList})` | 通过云对象批量删除云存储文件 |

**权限设置：**
- 公共权限：资源永久有效，无需 getTempFileURL
- 私有权限：资源有时间限制，需转换临时 URL

**自定义路径：** 可用 `dayjs().format("YYYYMMDD") + "/" + Date.now() + ".jpg"` 生成结构化的云存储路径。

### 5.8 uni-id 用户体系（预留）

**配置位置：** `uni-config-center/uni-id/config.json`

**核心配置项：**
- `passwordStrength` — 密码强度规则
- `tokenSecret` / `requestAuthSecret` — Token 加密密钥
- `tokenExpiresIn` — 客户端登录状态保持时间

**登录方式：**
- 微信小程序快捷登录（需条件编译区分）
- 账号密码登录
- 短信验证码登录（需开通 SMS 短信服务 + 模板配置）

**JQL 环境变量：** `$cloudEnv_uid` 可在 JQL 语句中直接获取当前用户 uid（依赖 uni-id）。

**迁移路径：** 当前用 `deviceId` 暂代用户标识，后续只需替换为 `openid`（来自 `clientInfo.uid`）。

### 5.9 触底加载

页面的 `onReachBottom` 生命周期钩子实现分页加载。本项目中的 classlist.vue 和 search.vue 均使用此机制：触底时 `pageNum++`，追加数据到列表。
