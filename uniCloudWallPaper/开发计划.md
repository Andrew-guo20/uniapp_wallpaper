## 阶段一：用户体系（2.5 天）

> **目标**：uni-id 微信登录可用的完整后端能力

### 1.1 配置 uni-id（0.5 天）

- [ ] 创建 `uni_modules/uni-config-center/uni-id/config.json`
- [ ] 微信小程序 OAuth 配置（AppID + AppSecret）
- [ ] Token 有效期 2h + 刷新阈值 10min
- [ ] 上传部署 uni-id-common 公共模块
- [ ] 调用 uni-id 登录接口验证可用

### 1.2 Schema 字段迁移（0.25 天）

- [ ] `wallpaper-user-score` 新增 `uid` 字段（可选，迁移期并存）
- [ ] `wallpaper-download-record` 新增 `uid` 字段
- [ ] `wallpaper-list` 新增 `favoriteCount` 字段（默认 0）

### 1.3 云对象 `_before` 改造（0.25 天）

- [ ] 提取 `clientInfo.uid`（uni-id 登录后有值）
- [ ] 保留 `this.deviceId` 做 fallback
- [ ] 评分/下载写入时同时写 `uid` + `deviceId`

### 1.4 用户查询兼容（0.25 天）

- [ ] `getUserInfo`：uid 优先，OR 查询覆盖迁移期
- [ ] `getUserWallList`：download/score 类型支持 uid + deviceId OR 查询
- [ ] `setupScore` / `downloadWall`：双写 uid + deviceId

### 1.5 数据迁移脚本（0.5 天）

- [ ] 新增云对象方法 `migrateDeviceToUid()`
- [ ] 逻辑：用户首次登录 → 查找 deviceId 历史记录 → 补充 uid
- [ ] 幂等：可重复执行不产生脏数据
- [ ] 迁移前自动备份（导出 JSON）

### 1.6 Schema 权限升级（0.25 天）

- [ ] `user-score`：create 条件 `auth.uid != null`
- [ ] `download-record`：create 条件 `auth.uid != null`
- [ ] 内容表权限不变（read=true, write=false）

### 1.7 登录相关云对象方法（0.5 天）

- [ ] 封装 `userLogin(code)` — 调 uni-id 登录 + 返回 token + 触发迁移
- [ ] 封装 `getUserProfile()` — 获取当前用户微信信息
- [ ] 封装 `checkToken()` — token 有效性校验

## 阶段三：互动功能（2 天）

> **目标**：收藏 + 评论 + 推荐 + 搜索 + 投稿后端方法全部就绪

### 3.1 收藏模块 `favorite.js`（0.5 天）

- [ ] `addFavorite({wallId})` — 唯一索引防重复 + `favoriteCount` inc
- [ ] `removeFavorite({wallId})` — 同时 dec `favoriteCount`
- [ ] `toggleFavorite({wallId})` — 智能切换（先查再决定 add/remove）
- [ ] `isFavorited({wallIds})` — 批量查询返回 Map
- [ ] `getUserWallList` 支持 `type=favorite`（lookup 联表）
- [ ] `getUserInfo` 返回 `favoriteSize`

### 3.2 评论模块 `comment.js`（0.5 天）

- [ ] `addComment({wallId, content})` — 敏感词过滤 + 频率限制（5条/分钟）
- [ ] `getComments({wallId, pageNum, pageSize})` — status=1 分页
- [ ] `deleteComment({commentId})` — 只能删自己的

### 3.3 推荐模块 `recommend.js`（0.25 天）

- [ ] `getRecommendWalls({pageSize=9})` — 标签权重推荐算法
- [ ] 冷启动兜底：随机高评分壁纸
- [ ] 排除已浏览/下载/收藏的壁纸

### 3.4 投稿模块 `upload.js`（0.25 天）

- [ ] `submitUpload({classid, description, tabs, picurl, smallPicurl})`
- [ ] 接收前端上传的图片 URL + 元数据 → 写入投稿表 status=0
- [ ] `getMyUploads({pageNum, pageSize})` — 我的投稿 + 状态

### 3.5 搜索增强方法（0.25 天）

- [ ] `getHotSearchKeywords()` — 近 7 天搜索聚合 TOP 10
- [ ] `getSearchHistory()` — 用户云端搜索历史
- [ ] `clearSearchHistory()` — 清除历史
- [ ] 搜索词统计写入（在 `searchWall` 中埋点）

### 3.6 部署验证（0.25 天）

- [ ] 上传部署所有模块
- [ ] 前端 API 对接测试（12 原有 + 11 新增 = 23 方法）
